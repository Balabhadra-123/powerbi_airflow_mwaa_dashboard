Major Step 2 — Data Model & SQL Measures
Since we're on DirectQuery, all calculations must live in SQL — no calculated columns. We'll use native SQL query folding via Power Query parameters.

2.1 — Create a SQL view in PostgreSQL
Run this in your PostgreSQL database so Power BI always queries a clean, pre-joined view:
sqlCREATE OR REPLACE VIEW vw_dag_dashboard AS
SELECT
    d.dag_name,
    d.dag_status,
    d.dag_start_time,
    d.dag_end_time,
    CAST(d.dag_start_time AS DATE)                                        AS run_date,
    EXTRACT(EPOCH FROM (d.dag_end_time - d.dag_start_time)) / 60.0       AS duration_minutes,
    DATE_PART('dow', d.dag_start_time)                                    AS day_of_week,
    CASE
        WHEN EXTRACT(EPOCH FROM (d.dag_end_time - d.dag_start_time)) / 60.0
             > m.sla_minutes THEN 'SLA Breached'
        ELSE 'Within SLA'
    END                                                                   AS sla_status,
    m.source_system,
    m.dag_type,
    m.sla_minutes
FROM dag_run_audit d
INNER JOIN dag_source_mapping m        -- only mapped DAGs come through
    ON d.dag_name = m.dag_name
WHERE m.is_active = 1
  AND d.dag_start_time >= CURRENT_DATE - INTERVAL '90 days';

The INNER JOIN here is the key — it automatically filters out any DAG not in your mapping table, so the Excel controls what appears on the dashboard.


2.2 — Point Power BI to the view
Go back to Power Query → Advanced Editor → replace your earlier SQL with simply:
sqlSELECT * FROM vw_dag_dashboard
This keeps Power BI clean and all logic centralised in PostgreSQL.

2.3 — Create a Date table for the 7-day bar chart
In Power Query → New Source → Blank Query → Advanced Editor → paste:
sqlSELECT DISTINCT
    CAST(dag_start_time AS DATE) AS run_date
FROM vw_dag_dashboard
ORDER BY run_date DESC
LIMIT 7
Name this query Last7Days. This will drive the bar chart X-axis cleanly.

2.4 — KPI measure queries (as separate PostgreSQL native queries)
Create these as individual queries in Power BI (each via Get Data → PostgreSQL) so they fold correctly under DirectQuery:
Total runs today:
sqlSELECT COUNT(*) AS total_runs
FROM vw_dag_dashboard
WHERE run_date = CURRENT_DATE
Completed today:
sqlSELECT COUNT(*) AS completed_runs
FROM vw_dag_dashboard
WHERE run_date = CURRENT_DATE
  AND dag_status = 'COMPLETED'
Failed today:
sqlSELECT COUNT(*) AS failed_runs
FROM vw_dag_dashboard
WHERE run_date = CURRENT_DATE
  AND dag_status = 'FAILED'
Currently running:
sqlSELECT COUNT(*) AS running_runs
FROM vw_dag_dashboard
WHERE dag_status = 'STARTED'
  AND run_date = CURRENT_DATE
SLA breaches today:
sqlSELECT COUNT(*) AS sla_breaches
FROM vw_dag_dashboard
WHERE run_date = CURRENT_DATE
  AND sla_status = 'SLA Breached'
Delta vs yesterday (for trend arrows on KPI cards):
sqlSELECT
    SUM(CASE WHEN run_date = CURRENT_DATE     THEN 1 ELSE 0 END) AS today_runs,
    SUM(CASE WHEN run_date = CURRENT_DATE - 1 THEN 1 ELSE 0 END) AS yesterday_runs,
    SUM(CASE WHEN run_date = CURRENT_DATE     THEN 1 ELSE 0 END)
  - SUM(CASE WHEN run_date = CURRENT_DATE - 1 THEN 1 ELSE 0 END) AS delta
FROM vw_dag_dashboard
WHERE run_date >= CURRENT_DATE - 1
